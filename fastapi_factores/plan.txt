### 2. FDA (Factor de Demanda Ajustada)

**Algoritmo Optimizado con pandas + numpy** (líneas 2102-2312 legacy):
```python
import pandas as pd
import numpy as np

# Fase 1: Obtener datos clusterizados en DataFrame
df = pd.DataFrame(consultar_medidas_con_factores_aplicados(...))

# Fase 2: Normalización FDA con pandas
1. Agrupar por fecha: df_grouped = df.groupby('fecha')
2. Sumar periodos: sumas = df_grouped[columnas_p].sum()
3. Calcular promedios: promedios = sumas.mean()
4. Calcular ajuste: ajuste = 1.0 - promedios.sum()
5. Encontrar máximos: maximos = sumas.max()
6. Aplicar ajuste vectorizado:
   df_ajustado = sumas.apply(lambda row:
       [max_val + ajuste if val == max_val else val
        for val, max_val in zip(row, maximos)]
   )
7. Redondear: df_ajustado = df_ajustado.round(5)
8. Validar suma: assert abs(df_ajustado.sum().sum() - 1.0) < 1e-9

# Repetir independientemente para: ORDINARIO, SABADO, FESTIVO
```

**Ventajas**:
- Operaciones vectorizadas (100x más rápido en datasets grandes)
- Código más conciso y legible
- Validaciones integradas de pandas

**Endpoint**: `POST /factores/calculos/fda`

**Precisión**: `round(valor, 5)` compatible con legacy.

### 3. FDP (Factor de Demanda Pronóstico)

**Algoritmo Optimizado con pandas + numpy** (líneas 3124-3147 legacy):
```python
import pandas as pd
import numpy as np

# Crear query SQL que trae A y R en una sola consulta con UNION
1. Consultar medidas A y R juntas (nueva query optimizada)
2. Cargar en DataFrame: df = pd.DataFrame(medidas)
3. Separar activas y reactivas:
   df_a = df[df['flujo'].str.startswith('A')]
   df_r = df[df['flujo'].str.startswith('R')]
4. Merge por fecha: df_merged = pd.merge(df_a, df_r, on='fecha', suffixes=('_a', '_r'))
5. Calcular FDP vectorizado para todos los periodos:
   for i in range(1, 25):
       P = df_merged[f'p{i}_a']
       Q = df_merged[f'p{i}_r']
       df_merged[f'fdp_p{i}'] = np.where(
           P == 0,
           np.where(Q == 0, 1.0, 0.0),  # Casos especiales
           np.cos(np.arctan(Q / P))      # Cálculo normal
       )
6. Redondear: df_merged = df_merged.round(5)

# Repetir para: ORDINARIO, SABADO, FESTIVO
```

**Ventajas**:
- Una sola query SQL para A y R (más eficiente)
- numpy.arctan + numpy.cos (vectorizado, ~50x más rápido)
- numpy.where maneja división por cero elegantemente
- Código más limpio

**Endpoint**: `POST /factores/calculos/fdp`

**Librerías**: `numpy` (agregar a requirements.txt)

## Funciones Auxiliares Clave

### En `calculos_service.py`

```python
import pandas as pd
import numpy as np

# Clustering con pandas
def _agrupar_medidas_clusterizadas(df: pd.DataFrame) -> pd.DataFrame:
    """Agrupa por barra+fecha y suma periodos"""
    columnas_p = [f'p{i}' for i in range(1, 25)]
    return df.groupby(['barra', 'fecha'])[columnas_p].sum().round(5)

# Normalización FDA con pandas/numpy
def _calcular_fda_normalizado(df: pd.DataFrame) -> pd.DataFrame:
    """Aplica algoritmo FDA completo usando operaciones vectorizadas"""
    # Suma y promedios vectorizados
    sumas = df.sum()
    promedios = df.mean()
    ajuste = 1.0 - promedios.sum()

    # Encontrar máximos y aplicar ajuste vectorizado
    maximos = df.max()
    df_ajustado = df.apply(lambda row:
        np.where(row == maximos, maximos + ajuste, row),
        axis=1
    )
    return df_ajustado.round(5)

# FDP con numpy
def _calcular_fdp_vectorizado(df_a: pd.DataFrame, df_r: pd.DataFrame) -> pd.DataFrame:
    """Calcula FDP para todos los periodos usando numpy"""
    df_merged = pd.merge(df_a, df_r, on='fecha', suffixes=('_a', '_r'))

    for i in range(1, 25):
        P = df_merged[f'p{i}_a'].values
        Q = df_merged[f'p{i}_r'].values

        # Vectorizado con numpy.where para manejar división por cero
        df_merged[f'fdp_p{i}'] = np.where(
            P == 0,
            np.where(Q == 0, 1.0, 0.0),
            np.cos(np.arctan(Q / P))
        )

    return df_merged[[f'fdp_p{i}' for i in range(1, 25)]].round(5)

# Conversión DataFrame ↔ JSON
def _df_to_response(df: pd.DataFrame, tipo_dia: str) -> Dict:
    """Convierte DataFrame a estructura de respuesta JSON"""
    return {
        "tipo_dia": tipo_dia,
        "n_registros": len(df),
        "factores": df.to_dict('index'),
        "suma_total": round(df.sum().sum(), 5)
    }
```

### Nuevas Queries Optimizadas (en `factores_service.py`)

Crear queries optimizadas que traigan datos ya pre-procesados:

```python
def consultar_medidas_con_factores_aplicados(
    fecha_inicial: str,
    fecha_final: str,
    mc: str,
    barra: str,
    tipo_dia: str,
    flujo_tipo: str
) -> List[Dict]:
    """
    Query optimizada que trae medidas con factores YA multiplicados usando SQL.
    Evita procesar en Python, usa el poder de PostgreSQL.
    """
    sql = """
    SELECT
        b.barra,
        m.fecha,
        m.codigo_rpm,
        m.flujo,
        a.factor,
        (m.p1 * a.factor) as p1,
        (m.p2 * a.factor) as p2,
        ... hasta p24
    FROM medidas m
    INNER JOIN agrupaciones a ON m.codigo_rpm = a.codigo_rpm AND m.flujo = a.flujo
    INNER JOIN barras b ON a.barra_id = b.id
    WHERE b.barra = %(barra)s
      AND m.fecha >= %(fecha_inicial)s
      AND m.fecha <= %(fecha_final)s
      AND b.mc = %(mc)s
      AND SUBSTRING(m.flujo, 1, 1) = %(flujo_tipo)s
      [filtros de tipo_dia]
    ORDER BY m.fecha
    """
```

**Ventaja**: PostgreSQL hace la multiplicación, Python solo agrega.

## Validaciones Críticas

1. **Rango de fechas**: fecha_final >= fecha_inicial
2. **Existencia de barra**: Validar con `consultar_barra_nombre()`
3. **Medidas disponibles**: Validar que existan datos en el rango
4. **División por cero**: Manejar en FDP cuando P=0
5. **Suma FDA**: Validar que resultado sume exactamente 1.0 (±1e-9)
6. **Tipos de flujo válidos**: Validar que la barra tenga medidas del tipo solicitado (A/R)

## Consideraciones Técnicas

### Precisión Numérica
- Legacy usa `Math.Round(valor, 5)` (C#)
- Python: `round(valor, 5)`
- Comparación de floats: usar epsilon `1e-9`

### Performance y Optimización
- **Usar pandas para manipulación de datos**: Más eficiente y legible
- **Usar numpy para operaciones vectorizadas**: Cálculos matemáticos más rápidos
- **Queries SQL optimizadas**: Crear queries específicas que traigan datos ya agrupados cuando sea posible
- **Sin límite arbitrario de días**: El usuario puede procesar rangos grandes si lo necesita
- **Caché de factores**: Evitar consultas repetidas para la misma barra

### Manejo de Errores
```python
try:
```
